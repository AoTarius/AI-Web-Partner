# L1 - 项目架构文档

## 项目概述
AI Partner Training - Vite + React + TailwindCSS v4 应用

## 技术栈
- **构建工具**: Vite 5.x
- **框架**: React 18.3
- **样式**: TailwindCSS v4 (Vite Plugin)
- **动效**: Framer Motion
- **图标**: Lucide React + React Icons
- **工具**: clsx, tailwind-variants

## 架构原则
- 组件化设计
- 响应式布局
- 性能优先
- 类型安全
- **设计系统驱动：一切设计必须来自设计系统**

## 🎨 设计系统约束（核心原则）

**重要：整个应用的所有视觉设计必须严格遵循设计系统规范。**

### 必须遵守的规则

1. **颜色来源**
   - 所有颜色必须来自设计系统调色板
   - 禁止使用任意颜色值（如 #FF5733）
   - 使用 TailwindCSS 预定义颜色类（slate, violet, fuchsia 等）

2. **组件来源**
   - 所有 UI 组件必须使用 `/components/ui/` 中的设计系统组件
   - 禁止创建未经设计系统定义的样式变体
   - 新组件必须先在设计系统中定义

3. **间距和尺寸**
   - 使用 TailwindCSS 标准间距系统（0.5, 1, 2, 4, 6, 8, 12, 16...）
   - 禁止使用任意值（如 `w-[237px]`）

4. **排版**
   - 遵循设计系统定义的字体大小和行高
   - 使用语义化的文本样式类

### 设计系统优先级

```
设计系统组件 > 自定义组件
系统颜色 > 任意颜色
标准间距 > 任意间距
```

## 目录结构
```
/src
  /components    # L2 模块级组件
  /pages         # L2 页面组件
  /layouts       # L2 布局组件
  /hooks         # L3 自定义 Hooks
  /utils         # L3 工具函数
  /assets        # 静态资源
```

## 开发流程
1. L1 定义架构和规范
2. L2 实现功能模块
3. L3 构建原子组件

## 分型结构说明
- **L1**: 系统级架构、全局配置、核心理念
- **L2**: 模块级功能、页面布局、业务逻辑
- **L3**: 组件级实现、工具函数、细节处理
